1.1 What is CPQ?

Complete Definition

CPQ (Configure Price Quote) is an enterprise software solution that automates three critical sales functions:

  • CONFIGURE: Guide customers through product selection and customization
  • PRICE: Automatically apply complex pricing rules, discounts, and bundling logic
  • QUOTE: Generate professional, approved quotes instantly

The Three Components in Detail

1. CONFIGURE - Product Selection

Purpose: Help sales reps and customers select the right products and options

Process:

  • Browse product catalog
  • Select base product
  • Choose options and add-ons
  • Handle product dependencies
  • Validate configuration

Example Configuration:

Base Product: Enterprise Software Version: Enterprise Edition License Type: Annual Subscription User Count: 100 users Support Tier: Premium 24×7 Add-ons Selected: - API Access: $500/year - Advanced Analytics: $1,000/year - Custom Integration: Included Training: On-site 3-day workshop Customizations: White-label, custom branding

2. PRICE - Automatic Calculation

Purpose: Apply business rules to calculate correct prices automatically

Process:

  • Retrieve list prices for selected products
  • Apply volume discounts
  • Apply bundle discounts
  • Apply customer-specific pricing
  • Apply promotional discounts
  • Calculate total and taxes

Example Pricing Calculation:

Line Items: ├─ Software (100 users × $500): $50,000 ├─ Premium Support (100 × $100): $10,000 ├─ API Access: $500 ├─ Advanced Analytics: $1,000 ├─ Training: $2,000 └─ Subtotal: $63,500 Pricing Rules Applied: ├─ Volume Discount (100+ users): -10% = -$6,350 ├─ Bundle Discount (Support+Analytics): -5% = -$2,925 ├─ Enterprise Customer: -5% = -$2,760 └─ Early Renewal Incentive: -2% = -$1,020 Final Pricing: ├─ Subtotal After Discounts: $50,445 ├─ Tax (8%): $4,036 └─ TOTAL: $54,481

3. QUOTE - Professional Document

Purpose: Create professional, branded quote documents

Features:

  • Professional formatting with company branding
  • Auto-populated with accurate data
  • Includes terms and conditions
  • Shows detailed line items
  • Displays applied discounts
  • Includes expiration date
  • Digital signature support

1.4 Core Concepts

Essential Terminology

Product: The base offering you sell (e.g., "Enterprise Software")

SKU: Stock Keeping Unit - unique identifier for each product variant

Bundle: Collection of products sold as a package with special pricing

Add-on: Optional products that enhance the base product

Configuration: Process of selecting and customizing products

Pricing Rule: Logic that applies discounts or price adjustments

Quote: Professional document showing configured products and prices

Line Item: Single product in a quote with quantity and price

Approval: Management review of quote before sending to customer

CPQ consists of several integrated components working together:

  • Product Catalog: All products, options, and add-ons

2.2 Pricing Models

Types of Pricing Models

1. Per-User Licensing

Model: Price per user/seat/license

Formula: Price per user × Number of users

Example: 100 users × $500/user = $50,000/year

Use Cases: SaaS, collaboration software, user seats

2. Usage-Based Pricing

Model: Price based on consumption/usage

Formula: Usage metric × Price per unit

Example: 1,000,000 API calls × $0.001 per call = $1,000

Use Cases: Cloud APIs, compute resources, data transfer

3. Tiered Pricing

Model: Different prices based on feature level

Formula: Selected tier × List price

Example: Standard ($500) vs Premium ($750) vs Enterprise ($1,000)

Use Cases: Subscription plans, service levels, feature bundles

4. Perpetual Licensing

Model: One-time purchase for unlimited use

Formula: License fee + ongoing support (optional)

Example: $50,000 one-time + $5,000/year support

Use Cases: Enterprise software, on-premise solutions

5. Subscription/Recurring

Model: Regular recurring payments (monthly, annual)

Formula: Monthly/annual fee × Number of months/years

Example: $500/month × 12 months = $6,000/year

Use Cases: SaaS, cloud services, support plans

6. Hybrid Pricing

Model: Combination of fixed + variable pricing

Formula: Base fee + (usage × rate) + (support × cost)

Example: $1,000 base + $0.10 per API call + $500 support

Use Cases: Complex enterprise deals

2.3 Pricing Rules

Pricing Rule Types

Volume Discount Rules

RULE: Volume Discount - Enterprise Software Condition: IF quantity of "Enterprise Software" >= 50 Action: APPLY discount of 10% on product Tier Structure: - 1-50 units: List price - 51-100 units: 10% discount - 101-250 units: 15% discount - 250+ units: 20% discount Example: - 50 units × $500 = $25,000 (no discount) - 75 units × $500 × 0.90 = $33,750 (10% off) - 200 units × $500 × 0.85 = $85,000 (15% off) - 300 units × $500 × 0.80 = $120,000 (20% off)

Bundle Discount Rules

RULE: Bundle Discount Condition: IF "Software" AND "Support" AND "Training" all selected Action: APPLY 5% discount on total Example: - Software: $50,000 - Support: $5,000 - Training: $2,000 - Subtotal: $57,000 - Bundle Discount (5%): -$2,850 - Total: $54,150

Customer Type Discount Rules

RULE: Enterprise Customer Discount Condition: IF Customer Type = "Enterprise" Action: APPLY 10% discount on total quote RULE: Partner Discount Condition: IF Account Type = "Partner" Action: APPLY 20% discount on products RULE: Competitor Buyout Condition: IF Replacing Competitor = true Action: APPLY 15% discount on first year

Promotional/Seasonal Rules

RULE: Year-End Promotion Condition: IF Date between 11/1 and 12/31 Action: APPLY 15% discount on all products RULE: Early Renewal Incentive Condition: IF Days Until Renewal <= 90 Action: APPLY 10% discount to encourage early renewal RULE: New Customer Condition: IF Customer Tenure = 0 (new) Action: APPLY 20% first-year discount

7.4 Interview Preparation

Common Interview Questions

Q1: Explain the three components of CPQ

Answer: Configure (product selection), Price (apply pricing rules), Quote (generate documents). Each component is essential for the complete solution.

Q2: How would you handle complex pricing?

Answer: Use pricing rules with conditions and tiers. Test thoroughly. Document business logic clearly. Use attributes for flexibility.

Q3: What's the most important part of quote approval?

Answer: Clear rules, appropriate authority levels, and automation. Balancing control with efficiency.
